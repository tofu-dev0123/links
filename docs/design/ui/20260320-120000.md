# UIデザインファイル

## 目的

リンクカードコンポーネントの単体テンプレートを確認する。
デフォルト状態・ホバー状態の2パターンを並べて表示し、コンポーネントの存在感・レイアウトを検証する。

## レイアウト

カード単体（案D: BC折衷）を横に2枚並べて表示。
- 左: デフォルト状態
- 右: ホバー状態（liftアニメーション適用後）

カードサイズ: 200×220px（正方形〜縦長）

## コンポーネント

### リンクカード（案D）

```
┌──────────────────┐
│              [↗] │  ← 右上: 外部リンクアイコン 16px（muted-foreground）
│                  │
│  [アイコン 64px]  │  ← 中央寄せ、プレースホルダー（Linkアイコン）
│                  │
│  サービス名       │  ← 左寄せ、font-weight: 600, 0.9rem
│  説明テキスト     │  ← 左寄せ、muted-foreground, 0.8rem
└──────────────────┘
```

- アイコン: Lucideスタイルのインラインプレースホルダー（Linkアイコン）、64px、primary色
- サービス名: "サービス名"（プレースホルダー）
- 説明テキスト: "ここに説明テキストが入ります"（プレースホルダー）
- 外部リンクアイコン: 右上固定、16px

## 状態・バリアント

- **デフォルト**: 通常表示（ボーダー: --border、影なし）
- **ホバー**: liftパターン適用（translateY(-4px)、box-shadow、border-color: --primary）

## 備考

- デザインシステムのカードパターンを縦長に再構成
- アイコンエリアは後から画像に置き換える想定のため、プレースホルダーで対応
- カード幅は200pxに変更（デザインシステムの340pxから縮小）
