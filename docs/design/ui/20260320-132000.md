# UIデザインファイル

## 目的

リンクページ全体のレイアウトをPC・モバイル両対応で確認する。
前回作成した案D（200×220px縦長カード）をベースに、プロフィールセクション＋カードグリッドで構成されたページ全体像を検証する。

## レイアウト

PC・モバイルの両レイアウトを横に並べて比較表示。

### PC版（max-width ~500px）

```
[T]
tofu
リンク集

[GitHub] [X      ]
[個人サイト] [note]
```

- カードグリッド: 2カラム（200px × 2 + gap 16px）
- ページ中央寄せ

### モバイル版（~320px）

```
[T]
tofu
リンク集

[GitHub   ]
[X        ]
[個人サイト]
[note     ]
```

- カードグリッド: 1カラム（200px、中央寄せ）

## コンポーネント

### プロフィールセクション

- アバター: "T"（ネイビー円形、80px）
- 名前: tofu
- bio: リンク集

### リンクカード × 4（案D）

| サービス   | アイコン（Lucide） | 説明文                         |
| ---------- | ------------------ | ------------------------------ |
| GitHub     | Github             | ソースコードを公開しています   |
| X          | Twitter            | 日々の気づきをつぶやいています |
| 個人サイト | Globe              | 個人サイトです                 |
| note       | FileText           | 記事を書いています             |

カード仕様（案D）:

- サイズ: 200×220px
- 右上: 外部リンクアイコン 16px（muted-foreground）
- 中央: サービスアイコン 64px（primary色）
- 左下: サービス名（font-weight: 600, 0.9rem） + 説明文（muted-foreground, 0.8rem）

## 状態・バリアント

- デフォルト表示のみ（ホバー状態は前回のモックで確認済み）

## 備考

- レスポンシブ: PC 2列 / モバイル 1列
- カードサイズは前回の案Dから変更なし（200×220px）
- アイコンは実装時に各サービスのブランドアイコンに置き換える想定
